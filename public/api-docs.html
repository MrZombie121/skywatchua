Документація Skywatch UA API

Основной сценарий: пользователь получает API-ключ в личном кабинете и встраивает карту в свой сайт через iframe. На внешнем сайте отображается только карта целей без основного интерфейса Skywatch UA.

Отримати API-ключ На головну карту

1. Отримання ключа

Відкрийте /account/api, зареєструйте акаунт або увійдіть в існуючий. Після входу створіть API-ключ у кабінеті.

2. Вбудовування карти

Рекомендований спосіб інтеграції:

<iframe
  src="https://your-domain.example/embed/map?api_key=YOUR_API_KEY"
  width="100%"
  height="640"
  style="border:0;border-radius:16px"
  loading="lazy"
  referrerpolicy="strict-origin-when-cross-origin"
></iframe>

Доступ до даних в embed-режимі відкривається тільки при валідному API-ключі.

3. Embed endpoint

GET /api/embed/events

  • Передача ключа: query-параметр api_key, заголовок x-api-key або Authorization: Bearer ....
  • Відповідь містить events, alarms, district_alarms, maintenance.
  • При невалідному ключі сервер повертає 401.
curl "https://your-domain.example/api/embed/events?api_key=YOUR_API_KEY"

4. Кабінет API

Доступні методи для авторизованого користувача:

  • POST /api/auth/register - реєстрація.
  • POST /api/auth/login - вхід.
  • POST /api/auth/logout - вихід.
  • GET /api/auth/me - поточний користувач і список ключів.
  • GET /api/auth/api-keys - список ключів.
  • POST /api/auth/api-keys - створення нового ключа.
  • DELETE /api/auth/api-keys/:keyId - відкликання ключа.
